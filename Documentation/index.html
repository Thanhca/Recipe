File Structure

Below is the structure of the recipe template:

  • /Recipe
    • /app
      • app.realease.apk(Recipe APK)
      • /src
        • /main
          • AndroidManifest.xml (App Manifest)
          • /java (App source code)
          • /xml (App xml code)
        • /test
        • /androidTest
    • /build
      • ... (System Properties)
    • /gradle
      • ... (System Properties)
    • /Documentation
      • /assets(Documentation resources)
      • index.html (App Documentation)
    • /gradlew (System file)
    • /gradlew.bat (System file)
    • /recipe.iml(System file)
    • /settings.gradle (System file)
    • /build.gradle (System file)
    • /gradle.properties (System file)

    Source code structure

    The source code are organized into folder for easy navigation:

    • / The root folder is where all the activity(ies) used resides
    • extra: This contains the three (3) extra pages designed for free for your usage
    • fragment: Contains all the fragment used in the app such as explore, my recipe, settings etc...
    • utility: Contains utility classes such as adapter classes for recyclieView, avatar image manipulation etc...

    How to test the app

    Testing the app is very easy, below are the steps you can use to test the template. If you're using a physical device (i.e phone) ensure it allows installation outside playstore.:

    • Android Studio: Import the full project (i.e recipe) into android studio and click on the run button at the top of the ide. Select the device of your choice and you're good to go

    • Physical Device: Send the apk in /app/app-realease.apk file to your mobile device, find it and click on it. Follow the instructions and you're good to go

    On the Login page just tap on the sign in button. This will take you to the dashboard so that you can explore the app
